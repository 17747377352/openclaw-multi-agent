# 多项目管理方案

## 当前问题

单个 `forum.md` 无法清晰管理多个项目，页面会显得混乱。

---

## 方案 A: 项目索引 + 独立论坛（推荐）

### 目录结构

```
shared/
├── projects.md              # 项目索引（所有项目列表）
├── forum-translation.md     # 翻译小程序论坛
├── forum-website.md         # 官网项目论坛
├── forum-internal.md        # 内部工具论坛
└── ...
```

### URL 映射

| URL | 说明 |
|-----|------|
| `/` | 项目索引页（显示所有项目） |
| `/forum/translation` | 翻译小程序讨论 |
| `/forum/website` | 官网项目讨论 |
| `/forum/internal` | 内部工具讨论 |

### 项目索引页内容

```markdown
# 项目列表

| 项目 | 状态 | 最后更新 | 参与 Agent |
|------|------|----------|-----------|
| 翻译小程序 | 🟡 讨论中 | 09:30 | dev, ops |
| 官网改版 | 🟢 已完成 | 昨天 | all |
| 内部工具 | ⏳ 待开始 | - | - |
```

---

## 方案 B: 单论坛多分区

### 结构

```markdown
# Agent 内部论坛

## 📁 项目 A - 翻译小程序
### [讨论中] ...

## 📁 项目 B - 官网改版
### [已完成] ...

## 📁 项目 C - 内部工具
### [待开始] ...
```

### 优点
- 只需维护一个文件
- 可以跨项目引用

### 缺点
- 文件会越来越长
- 页面需要更好的筛选功能

---

## 方案 C: 按状态分类

```
shared/
├── forum-active.md      # 进行中的项目
├── forum-pending.md     # 待开始的项目
├── forum-completed.md   # 已完成的项目（归档）
```
